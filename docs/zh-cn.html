快速安装

在 VS Code 中快速选择语音后端并开始生成

1

从 VS Code 市场安装

在 VS Code 扩展中搜索 "Speechify" (Ctrl+Shift+X) 或点击上方安装按钮。

2

选择语音后端

在 VS Code 设置中选择使用 Azure Speech，或切换到 CosyVoice（本地）/ Qwen3-TTS + MLX-Audio（本地）作为生成后端。

3

开始生成音频

选中文本后，可通过右键菜单或 Ctrl+Shift+P 运行 Azure：生成配音、本地 CosyVoice：生成配音 或 本地 Qwen3-TTS：生成配音。

本地声音克隆使用流程

在 VS Code 内完成声音克隆、参考录音和本地转录

1. 选择本地引擎

CosyVoice 通过本地 FastAPI 服务运行，默认读取 speechify.cosyVoice.baseUrl，默认值为 http://127.0.0.1:50000。

Qwen3-TTS 则直接通过 mlx-audio 和本地 Python 环境运行，使用 speechify.qwenTts.pythonPath 指定，不需要常驻服务。

2. 配置参考音色

使用 本地 CosyVoice：设置参考声音 或 本地 Qwen3-TTS：设置参考声音，可以直接在 VS Code 里录制参考音频，也可以选择已有音频或视频文件。

如果选择视频，Speechify 会先自动抽取音轨，再打开工作区设置，方便你检查和编辑参考文本。

3. 生成克隆语音

右键选中文本后，使用 本地 CosyVoice：生成配音 或 本地 Qwen3-TTS：生成配音。Speechify 会读取你配置的参考音频，并把生成结果接入当前项目工作流。

参考文本不是强制项，但强烈建议填写。无论是 CosyVoice 还是 Qwen3-TTS，参考音频加参考文本都会带来更好的克隆质量。

本地配置说明

参考媒体的转录完全在本地执行。在 macOS 上，Speechify 会优先尝试 Whisper MLX，以更好利用 Apple Silicon；如果不可用，会自动回退到本地 Python Whisper。

在工作区设置里，请按你实际使用的 provider 配置本地后端地址或 Python 路径、参考音频路径和参考文本。

每个 CosyVoice / Qwen3-TTS 配置项的键名、用途说明，以及完整的 settings.json 示例，已经放在项目 README 中。

本地引擎仓库地址：

  • CosyVoice — github.com/FunAudioLLM/CosyVoice
  • Qwen3-TTS（mlx-audio） — github.com/Blaizzy/mlx-audio
